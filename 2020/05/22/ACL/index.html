ACL(Access Control List)

No tag
Publish Date:   2020-05-22

1、ACL是一种包过滤技术

2、ACL基于IP包头的IP地址、四层TCP/UDP头部的端口号、【5层数据】
基于三层和四四层过滤
3、ACL在路由器上配置，也可以在防火墙上配置（一般称为策略）

5、ACL主要分为2大类：
1）标准ACL
2）扩展ACL

6、标准ACL：
表号：1-99
特点：只能基于源IP对包进行过滤
命令：
conf t
access-list 表号 permit/deny 源IP或者源网段 反子网掩码
注释：反子网掩码，将正子网掩码0和1倒置
255.0.0.0 – 0.255.255.255
反子网掩码作用：用来匹配，与0对应的需要严格匹配，与1对应的忽略

例如：access-list 1 deny 10.1.3.1 0.0.0.255
反子网掩码有3个0，所以严格匹配ip地址的前三位10.1.3，只要ip是10.1.3就符合要求

access-list 1 deny 10.1.3.1 0.0.0.0
如果有4个0就是down掉具体的ip
简写：access-list 1 deny host 10.1.3.1

access-list 1 deny 0.0.0.0 255.255.255.255
如果有4个255就是down掉所有人
简写：access-list 1 deny any

查看ACL表：
show ip access-list [表号] (表少的话不用加表号直接show）

将ACL应用到接口：
int f0/x
ip access-group 表号 in/out
exit

7、扩展ACL：
表号：100-199
特点：可以基于源IP、目标IP、端口号、协议等对包进行过滤

access-list 100 permit/deny 协议 源IP或者源网段 反子网掩码 目标IP或者源网段 反子网掩码 [eq 端口号]

eq的意思是等于=

注释：协议：tcp/udp/icmp/ip （tcp和udp搭配端口号）
案例：
acc 100 permit tcp host 10.1.1.1 host 20.1.1.3 eq 80
acc 100 deny ip host’ 10.1.1.20 20.1.1.0 0.0.0.255
acc 100 permit ip any any

deny 拒绝
permit

8、ACL原理
1）ACL表必须应用到接口的进或者出反向才生效
2）一个接口的一个方向只能应用一张表
3）进还是出方向的应用是取决于流量控制的总方向
4）ACL表是严格自上而下检查每一条，所以要注意书学顺序
5）每一条是由条件和动作组成，当流量完成满足条件则放行，当某流量没有满足某条件时，则继续检查下一条
6）标准ACL尽量写在靠近目标的地方
7）原理：
1、做流量控制，首先要先判断ACL写的位置（哪个路由器？哪个接口哪个方向）
2、再考虑怎么写ACL
3、如何写
首先判断最终要允许所有还是拒绝所有
然后写的时候要注意：将严格的控制写在前面

8）一般情况下，标准或者扩展ACL一旦编写好，无法修改某一条，也无法删除某一条，也无法修改顺序。
也无法往中间插入新的条目，只能一直在最后添加新的条目。
如想修改或者插入、删除，只能删除整张表重新写。
命令：
conf t
no access-list 表号

查看ACL表：
show ip access-list [表号] (表少的话不用加表号直接show）

9、面名ACL：
作用：可以对标准或者扩展ACL进行自定义命名
优点：自定义命名更加容易辨认，也便于记忆
可以任意修改某一条，或者删除某一条，也可以往中间插入某一条
命令：
conf t
ip access-list standard/extended 自定义表面
开始deny或者permit编写ACL条目
exit

删除某一条：
ip access-list standard/extended 自定义表名
no 条目 ID
exit

插入某一条：
ip access-list standard/extended 自定义表名
条目 ID 动作 条件
exit

SW3-1(config)#ip access-list extended cw
SW3-1(config-ext-nacl)#den
SW3-1(config-ext-nacl)#deny ip 192.168.1.0 0.0.0.255 host 192.168.3.1
SW3-1(config-ext-nacl)#deny ip 192.168.2.0 0.0.0.255 host 192.168.3.1
SW3-1(config-ext-nacl)#deny ip 192.168.4.0 0.0.0.255 host 192.168.3.1
SW3-1(config-ext-nacl)#per
SW3-1(config-ext-nacl)#permit ip any hos
SW3-1(config-ext-nacl)#permit ip any host 192.168.3.1
SW3-1(config-ext-nacl)#do show ip acc
